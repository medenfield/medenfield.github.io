Hello world!

This is my a personal project and portfolio site where I intend to write about new things I’m learning about programming and code. I will also revisit things I’ve created that I’m proud about and want to display publicly. Much of what is contained will be about challenging myself to accomplish X using something new, and then writing about it.

For this site, I went with Hugo - a static site generator written in Go. One my friends and mentors happened to be playing with Hugo that morning as well, so that was the inspiration there. It was very quick to set up. I explored using Firebase to host the site, but really that’s overkill and it’s easy enough to just host the whole thing on Github for free (and show any related code to boot).

It is very easy to get Hugo up and running and the tutorials are great. The thing I like most about Hugo is that it listens for changes as the server runs, meaning what I’m writing is getting updated in real time as I save the file, without me refreshing the page. Of course that’s all just happening locally, but over time, it will be easy enough to automate that process to build directly from my github page using Jenkins.

Hugo is configured to generate HTML from Markdown files. I haven’t had a chance to write a bunch of markdown so far; this post is pretty much a first past. I’m following the examples at Ghost Blog and yeah, super simple. Also, I installed the Sublime markdown extended package with the Monokai Theme for writing markdown in Sublime Text with a more colorful and highlighted editing experience.
